# 0G Labs Integration

0G (Zero Gravity) is three separate things that share infrastructure but work independently: Compute
(LLM inference), Storage (decentralized file storage), and Chain (an EVM-compatible blockchain for
smart contracts). This confused me for a while reading the docs because they all live under the same
brand and the documentation doesn't always make clear which one it's talking about.

---

## 0G Compute

This is where the LLM inference happens. You initialize the `@0glabs/0g-serving-broker` SDK with
a private key and a chain RPC URL, and it handles the on-chain payment channel mechanics
automatically. After that, you make requests using an OpenAI-compatible chat completions API.

```typescript
const broker = await createZGServingNetworkBroker(signer);
await broker.listService();  // confirms connection

const response = await broker.generateAnswer(
  serviceName,
  modelName,
  messages,
  null
);
```

The testnet model is `qwen-2.5-7b-instruct`. Response quality at 7B is acceptable for structured
vulnerability analysis when you use targeted system prompts — the model doesn't have broad security
knowledge baked in the way a larger model would, but if you tell it exactly what to look for in the
code, it finds things.

Latency on testnet runs around 8-12 seconds per call. Four agents auditing in parallel means the
total audit time is roughly as long as the slowest single call, not four times longer — which is
the point of the parallel architecture.

Billing goes through payment channels that the broker manages on 0G Chain. You need testnet OG
tokens before the first inference call. The faucet is at `faucet.0g.ai`. Rate limits aren't fully
documented so I budgeted conservatively.

## 0G Storage

File storage with content-addressed retrieval. You upload a blob and get back a `rootHash`, which
serves as the permanent identifier for that content. You can retrieve it later by rootHash from the
indexer.

```typescript
const indexer = new Indexer(ZG_STORAGE_INDEXER_URL);
const [tree, err] = await Blob.fromBuffer(Buffer.from(data)).merkleize();
const tx = await indexer.upload(tree, ...);
// rootHash from the transaction receipt
```

The testnet indexer is at `indexer-testnet.0g.ai`. In practice the testnet indexer can be slow or
temporarily unresponsive. The audit pipeline handles this gracefully: if the upload fails, the
report is cached to disk locally and the attestation still proceeds with a local hash. The on-chain
record notes the degraded state.

SDK is `@0gfoundation/0g-ts-sdk`. The documentation is thin and took some time to work through —
most of what I know about the actual upload flow came from reading the SDK source rather than the
docs.

## 0G Chain

An EVM-compatible blockchain at chain ID `16602`. You deploy contracts with Foundry targeting this
chain, and interact with them via ethers.js with an RPC URL pointing to the 0G testnet.

```typescript
const provider = new ethers.JsonRpcProvider(ZG_CHAIN_RPC);
const signer = new ethers.Wallet(privateKey, provider);

// Write the audit attestation after storing the report
const attestation = new ethers.Contract(
  AUDIT_ATTESTATION_ADDRESS,
  AuditAttestation.abi,
  signer
);
await attestation.attest(contractAddr, findingsHash, storageRootHash, critCount, highCount);
```

The `AgentRegistry` and `AuditAttestation` contracts are deployed here. Anyone can verify an
audit by taking the `storageRootHash` from the on-chain attestation record and retrieving the
full report from 0G Storage.

Block explorer: `chainscan-galileo.0g.ai`. Transaction confirmation is fast on testnet, usually
under 5 seconds.

## In AgentMesh

The three 0G services fit together as a pipeline:

1. **0G Compute** — four agents call it in parallel to analyze the contract for their specialty
2. **0G Storage** — the final consensus report gets uploaded; rootHash returned
3. **0G Chain** — `attest()` is called with the rootHash and severity breakdown, creating a
   permanent on-chain record linking the audited contract address to the stored report

The proof page in the frontend shows both the 0G Chain attestation transaction hash and the 0G
Storage rootHash so judges can independently verify the pipeline ran end-to-end.
